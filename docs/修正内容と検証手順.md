# YMMKeyboard 修正内容と検証手順（2026-05-29）

## 1. 今回の修正内容

### 1-1. RP2040 エンコーダの P/R 対応
- 対象: `firmware/src/RP2040ZeroCode/code.py`
- 変更内容:
  - エンコーダ入力（`SW_36`, `SW_37`）を `P` のみ送信から、`P -> R` の連続送信へ変更。
  - 1ノッチを「疑似タップ」として扱うようにした。
- 目的:
  - プラグイン側の押下状態が解除されず残る問題（張り付き）を防ぐ。

### 1-2. タイムライン操作の UI スレッド実行
- 対象: `ymm-plugin/src/YMMKeyboardPlugin/Actions/KeyboardAction.cs`
- 変更内容:
  - `PlusSeekFrame` / `MinusSeekFrame` で `Application.Current.Dispatcher.Invoke(...)` 経由で `CurrentFrame` を更新。
- 目的:
  - シリアル受信スレッドから直接 UI/Timeline を触ることによる不安定動作リスクを低減。

### 1-3. COM 切断時の状態クリア
- 対象:
  - `ymm-plugin/src/YMMKeyboardPlugin/SerialKeyboard/SerialKeyboardLink.cs`
  - `ymm-plugin/src/YMMKeyboardPlugin/Key/Keymacro.cs`
- 変更内容:
  - `SerialKeyboardLink` に `KnownUids` を追加。
  - `DisconnectPort` 実行時に、該当 UID の押下状態・コンボ消費状態・遅延実行待ちをクリア。
  - UID ごとのマクロキャッシュも削除して再接続時に再構築。
- 目的:
  - 切断前状態の残留による誤動作（誤コンボ、意図しない実行）を防ぐ。

## 2. 手動検証手順（実機）

### 2-1. エンコーダ張り付き確認
1. `firmware/src/RP2040ZeroCode/code.py` を書き込み、実機を接続。
2. YMM 側で `SW36` または `SW37` に単発アクションを割り当てる。
3. エンコーダを1ノッチ回す。
4. 同じ方向へ連続して数回回す。
5. 期待結果:
   - 回した回数分だけアクションが発火。
   - 押下状態が残って以降のキーが誤動作しない。

### 2-2. スレッド境界確認（再現負荷）
1. `PlusSeekFrame` / `MinusSeekFrame` を複数キーに割り当てる。
2. 連打・同時入力を繰り返す。
3. 期待結果:
   - シークが継続動作し、UI 例外や停止が発生しない。

### 2-3. 切断/再接続確認
1. キー押下中または連続入力中に COM 切断（または機器抜き差し）。
2. 再接続して同じキー操作を実行。
3. 期待結果:
   - 以前の押下状態が残らず、通常どおり単押し/コンボ判定される。

## 3. テストプログラム（擬似イベント）

- 追加: `tools/KeyboardEventStateTester`
- 目的:
  - `P/R` と単押し遅延・コンボキャンセルの基本ロジックを機械的に確認する。
- 実行:
  - `dotnet run --project tools/KeyboardEventStateTester/KeyboardEventStateTester.csproj`
- 成功時:
  - 各ケースが `[OK]` 表示され、最後に `All test cases passed.` が出る。
