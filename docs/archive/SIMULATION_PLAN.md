# SIMULATION_PLAN

## 判定基準

- `Wokwi対応`: Wokwi 上で十分に再現できる
- `一部対応`: 入力ロジックや GPIO は再現できるが、実機と同一にはならない
- `実機必須`: PC 側 USB 連携や物理特性に依存する

## 機能別判定

| 対象 | 判定 | 理由 |
|---|---|---|
| GPIO | 一部対応 | 行列スキャンやボタン入力の論理検証は可能だが、実配線確認が必要 |
| OLED | 一部対応 | もし追加するなら描画確認は可能だが、現行実装では未使用 |
| WS2812 | 一部対応 | タイミング検証の代替は可能だが、実機での発光確認が必要 |
| USB HID | 実機必須 | YMM4 とのホスト側連携が必要で Wokwi だけでは再現できない |
| USB CDC | 実機必須 | ホスト PC とのシリアル接続確認が必要 |
| PIO | 一部対応 | ロジック検証は可能だが、実際の周辺機器接続確認は実機が必要 |
| ADC | 一部対応 | 値の読み取りロジックは再現できるが、実アナログ特性は別途確認が必要 |
| PWM | 一部対応 | デューティ比のロジック確認は可能だが、実機の負荷条件が必要 |

## 現行プロジェクトでの実態

- 現行の中心機能は `USB HID` と `USB CDC` に依存しているため、最終確認は実機が必要です。
- `RP2040ZeroCode` の行列スキャンやエンコーダ入力は Wokwi で論理確認はできますが、PC 側との end-to-end は実機必須です。
- `RP2040TinyUsb` は TinyUSB の動作確認までは実機で行う必要があります。

## Wokwi に向く部分

- 行列スキャンの入力遷移
- エンコーダの A/B 位相
- 単純な GPIO デバウンス
- スイッチ ID 変換ロジック

## Wokwi に向かない部分

- YMM4 への HID 入力
- YMM4 への CDC ログ
- Windows 側の COM/HID 列挙
- USB デバイス名や VID/PID の実機列挙

## 推奨シミュレーション手順

1. GPIO 入力を模した最小スケッチを作る
2. 行列スキャンの論理遷移を検証する
3. エンコーダの回転方向と `SW36/SW37` 変換を確認する
4. 実機で USB HID/CDC の end-to-end を確認する

## 結論

- **Wokwi だけで完結する範囲は限定的** です。
- **USB HID/CDC を伴う本体確認は実機必須** です。
