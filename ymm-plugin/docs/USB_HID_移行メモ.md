# USB HID移行メモ

## 目的
- COM接続の不安定さを減らし、USB HIDを中心に運用できるようにする。

## 現在の実装状況
- プラグイン設定に `接続モード` を追加
  - `自動`: HIDを開始しつつCOMも利用可能
  - `COM`: COMのみ
  - `HID`: COM接続要求を無視してHIDのみ
- `HID診断` ボタンを追加
  - 接続中HIDデバイスの VID/PID などをレポート出力
- HID入力リンクを追加
  - 入力レポートに `UID:P:SW_xx` 形式のASCIIが含まれる場合はイベントとして処理

## 設定手順
1. YMM設定のキーボードプラグインを開く
2. `HID診断` を押す
3. レポートを見て対象デバイスの `VID/PID` を確認
4. `VID(16進)` `PID(16進)` に入力して `適用`
5. `接続モード` を `HID` または `自動` に設定

## 注意点
- 現在の `firmware/src/RP2040ZeroCode/code.py` はシリアル出力中心です。
- HIDで確実にイベントを拾うには、ファーム側でもHIDレポートのイベント送信仕様を合わせる必要があります。
- まずは `自動` モードで実運用し、ログを見ながらHIDイベント形式を固めるのを推奨します。
